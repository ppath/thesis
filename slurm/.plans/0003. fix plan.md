## Changes

### 1. Epilog: scope the GPU process kill to the ending job's cgroup *(critical)*

File: `etc/slurm/epilog.sh`

Today, on job end, the epilog runs `nvidia-smi --query-compute-apps=pid` for
each allocated `/dev/nvidiaN` and `kill -9`s every PID it finds. Because shards
let multiple jobs share one physical GPU, this kills the *other* job's
processes whenever an unrelated job on the same GPU ends.

Fix: only kill PIDs whose `/proc/$pid/cgroup` still references
`job_${SLURM_JOB_ID}`. The job's normal proctrack/cgroup teardown already kills
in-cgroup tasks; this epilog is defensive cleanup for stragglers that
reparented to PID 1, so cgroup membership is the right filter and works on
both cgroup v1 and v2.

```bash
job_id="${SLURM_JOB_ID:-}"
gpus="${SLURM_JOB_GPUS:-}"
[[ -z "$gpus" || -z "$job_id" ]] && exit 0

for gpu in ${gpus//,/ }; do
    while read -r pid; do
        [[ -z "$pid" ]] && continue
        if grep -q "job_${job_id}\b" "/proc/$pid/cgroup" 2>/dev/null; then
            kill -9 "$pid" 2>/dev/null || true
        fi
    done < <(nvidia-smi --query-compute-apps=pid --format=csv,noheader \
                        --id="$gpu" 2>/dev/null)
    nvidia-smi -i "$gpu" -q -d MEMORY >> /var/log/slurm/epilog.log 2>&1 || true
done
exit 0
```

No prolog will be added.

### 2. Partitions: drop `cpu`, rename `gpu` ג†’ `main`

Files: `etc/slurm/slurm.conf`, `README.md`, `setup.sh`

- Delete the `PartitionName=cpu ג€¦` block; the cluster has one node and the
  `cpu` partition only created confusion (it didn't actually block GPU jobs).
- Rename `PartitionName=gpu ג€¦` to `PartitionName=main ג€¦`.
- On `main`, add `DefCpuPerGPU=8` and `DefMemPerGPU=64G` so a bare
  `--gres=gpu:h100:1` no longer gets 1 CPU + 4 GB.
- Lower `MaxMemPerCPU` from `16384` to `15700` so the cap is honest vs.
  `RealMemory גˆ’ MemSpecLimit` (502712 MB / 32 phys cores).
- Replace every `-p gpu` with `-p main` in the README examples and in the
  smoke test at the end of `setup.sh`.

### 3. `slurm.conf` cleanups

File: `etc/slurm/slurm.conf`

- Drop `gres/gpumem` from `AccountingStorageTRES` ג€” no `gpumem` GRES is
  declared, leaving it advertised is misleading.
- Set `InactiveLimit=0` (off) with a one-line comment that 30 min was
  surprising users walking away from interactive jobs.
- Tighten the `ReturnToService=1` comment: `0`=never, `1`=auto-return after
  non-responsive timeout once the node responds with matching config,
  `2`=also returns from admin-initiated DOWN. We use `1`.

`AccountingStorageEnforce=associations,limits,qos` is **not** loosened ג€” the
config is the source of truth; the README will be updated to match.

### 4. `cgroup.conf`: comment-only ג€” document v2 swap dependency

File: `etc/slurm/cgroup.conf`

Add a comment near `ConstrainSwapSpace=yes` noting that this only takes effect
when the kernel exposes swap accounting. On Ubuntu 22.04+ that's the default
under cgroup v2; on older kernels you need `swapaccount=1` on the kernel
command line. No code change.

### 5. `gres.conf`: comment-only ג€” shard `Cores=` cross-check

File: `etc/slurm/gres.conf`

Add a comment near the shard entries reminding the operator to run
`nvidia-smi topo -m` and `slurmd -C` once and verify that the manual shard
`Cores=` matches NVML's reported NUMA affinity, since BIOS enumeration sometimes
interleaves sibling threads. No code change.

### 6. `setup.sh` changes

File: `setup.sh`

- **Preflight check (new, top of script after `set -euo pipefail`)**: require
  `command -v nvidia-smi`, a non-empty `nvidia-smi -L`, and
  `ldconfig -p | grep -q libnvidia-ml`. On any miss, print the missing
  component **and the exact `apt install` command** that provides it, then
  `exit 1`. Warn (non-fatal) if `nvidia-persistenced` is not active.
- **Move `slurmd -C`** out of step 3 into a position immediately after the
  config-copy step. Capture output and echo it next to the relevant
  `slurm.conf` lines so the operator sees the comparison.
- **Idle poll before smoke test**: bounded 30 s loop on
  `sinfo -h -o '%T' -n enif | head -n1` until it reports `idle`, then run
  the smoke test.
- **Smoke test partition flag**: `srun -p gpu` ג†’ `srun -p main`.
- **Deploy logrotate file** (see below).

### 7. New file: `etc/logrotate.d/slurm`

Rotate `/var/log/slurm/*.log` (slurmctld, slurmd, slurmdbd, epilog) weekly,
keep 8 rotations, `compress`, `notifempty`, `missingok`, `copytruncate`.
Installed by `setup.sh` as `0644 root:root` to `/etc/logrotate.d/slurm`.

`HealthCheckProgram` is **not** added (deferred per user direction).

### 8. `README.md` updates

File: `README.md`

- Replace `-p gpu` with `-p main` everywhere; remove any mention of a `cpu`
  partition.
- Add a short **"Direct SSH access"** subsection: users may continue to
  `ssh enif` as normal; SLURM's cgroup limits only apply to processes started
  through `srun`/`sbatch`, and processes launched directly over SSH run
  outside any job cgroup with no CPU/memory/GPU isolation. We are intentionally
  not installing `pam_slurm_adopt`.
- Drop any wording that suggests dropping `limits,qos` from
  `AccountingStorageEnforce` ג€” the deployed config keeps them strict.
- "Adding a second node later": with `enable_configless` already set, the new
  node's `slurmd` can be started with `--conf-server=enif:6817`; only a
  *node-local* `gres.conf` is required. Adjust steps 3ג€“4 accordingly.
- Fix the claim that `cgroup.conf` "must be identical on every node": cgroup
  config is per-node; identical is fine in practice but not required.
- Add a one-line note that `MariaDB` is left at Ubuntu's default unix_socket
  auth on localhost only and should be reviewed before any second node points
  at this DBD over the network.
